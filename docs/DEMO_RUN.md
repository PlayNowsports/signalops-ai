## Important validation result

Twenty-six accounts have conflicting health or utilization values across open FY27 renewals. SignalOps surfaces this as a warning and selects one highest-priority opportunity row as the evidence basis for the account queue. It does not combine contradictory row labels as independent evidence.

Other supplied-file findings:

- 96 rows where `Age` and `Opp Age Recal` differ by more than 30 days;
- 67 rows where `Generated by = Partner` but `Partner of Record` is blank;
- 190 renewal rows where `Renewal ARR` and `Total Booking` differ by more than 15%.

These counts are data-quality observations across the full file, not all part of the current open renewal book.

## Demo capture checklist

For the submission recording, capture a 60–90 second run:

1. Open the live application and upload the supplied workbook.
2. Show the completed run banner with input hash and rule version.
3. Open Data quality and point to the conflicting-account warning.
4. Open a Critical account and show the evidence, source row, action owner, and human-review warning.
5. Create or edit a rule, enter a reason, and show the version increment.
6. Export the weekly queue and rule pack.

The recorded run should remain short. The panel needs evidence that the tool works, not a narrated product tour.
